# 桌面宠物「Desktop Cat」设计文档

- **日期**: 2026-06-10
- **状态**: 已确认,待生成实现计划

## 1. 目标

做一个常驻 Windows 桌面的「桌面宠物」程序:在桌面某处展示一个透明悬浮的宠物视频窗口,自动轮播用户指定本地文件夹里的宠物视频。窗口无边框、置顶、可拖动、尺寸可调,关掉再开记住上次的位置和尺寸。

用户提供的是**绿幕背景**的宠物视频;程序负责在导入时把绿幕抠掉、转成透明视频后再播放。

## 2. 核心决策(已与用户确认)

| 维度 | 决策 |
|---|---|
| 展示形态 | 透明悬浮窗,只显示宠物轮廓贴在桌面上 |
| 技术栈 | Electron + 原生 HTML/CSS/JS(不引前端框架) |
| 视频来源 | 本地文件夹,右键菜单用系统弹窗选择,选完记住 |
| 轮播方式 | 一个视频播完自动切下一个,最后一个播完回到第一个,无限循环 |
| 视频素材 | 用户提供**绿幕视频**(非透明视频) |
| 抠像时机 | **导入时**用 ffmpeg 一次性把绿幕转成带 alpha 的透明 webm,缓存复用;播放时零开销 |
| ffmpeg | 用 npm `ffmpeg-static` 打包,用户无需自行安装 |
| 窗口位置/尺寸 | 可拖动;尺寸通过右键菜单调(小/中/大档位 + 自定义数值);重启记住位置和尺寸 |
| 退出/控制 | 系统托盘图标 + 右键菜单,均含「退出」 |
| 开机自启 | 右键/托盘菜单里一个「开机启动」开关 |

## 3. 整体架构

技术栈:**Electron**(透明无边框置顶窗口 + `<video>` 播放 alpha 通道 webm 均开箱即用) + **ffmpeg-static**(打包的 ffmpeg,用于绿幕转透明)。不引入前端框架,核心逻辑简单,保持轻量。

### 进程划分(Electron 标准双进程)

| 进程 | 职责 |
|---|---|
| **主进程 `main.js`** | 创建透明窗口、系统托盘、右键菜单、扫描源文件夹、调用 ffmpeg 转换、读写配置(位置/尺寸/文件夹路径/自启)、注册开机自启。一切「系统层」的事。 |
| **渲染进程 `renderer`** | 只负责循环播放主进程递过来的**透明 webm** 列表,一个播完播下一个。纯播放器,完全不知道绿幕的存在。 |
| **`preload.js`** | IPC 安全桥接。`contextIsolation` 开启,不裸开 `nodeIntegration`。 |

**职责隔离原则**:绿幕 → 透明的所有处理都在 main 里一次性做完;renderer 永远只认透明 webm,只接收一个「视频 URL 列表」。两边可独立理解与测试。

## 4. 数据流

```
启动 → main 读配置(源文件夹 / 窗口位置尺寸)
     → main 扫描「已转换缓存目录」,得到透明 webm 列表 → IPC 发给 renderer
     → renderer 顺序循环播放(零开销)

右键「选择视频文件夹…」
     → main 弹系统对话框选绿幕源文件夹
     → main 后台用 ffmpeg 逐个把绿幕视频转成透明 webm,存到缓存目录
       (托盘/菜单显示「转换中 2/5…」,转完才推列表)
     → 已转换过的文件跳过(按文件名 + 修改时间判断,命中缓存不重复转)
     → 转换完成 → 推新列表给 renderer 开始播

右键「尺寸」→ main 改窗口大小 + 存配置
拖动窗口 → main 监听 move/resize → 防抖后存配置
```

### 关键设计点

- **源文件夹与缓存目录分离**:用户原始绿幕视频一个字节都不改;转换产物单独存在程序数据目录(`userData`)里的缓存子目录。
- **转换一次性**:同一视频转过即缓存,下次启动直接用,不重复转。
- **绿幕色值**:默认抠标准绿幕(`0x00FF00` 附近)。预留 `chromaColor` 配置项,暂不暴露 UI(YAGNI);若实际出现绿边再考虑加「灵敏度」微调。

## 5. 配置存储

JSON 文件,存于 Electron 标准用户数据目录(`app.getPath('userData')`,Windows 为 `%APPDATA%/desktop-cat/`),不污染工作目录。

| 配置项 | 内容 | 何时写 |
|---|---|---|
| `sourceFolder` | 用户选的绿幕视频源文件夹路径 | 选文件夹时 |
| `windowBounds` | 窗口 x / y / width / height | 拖动、改尺寸时(防抖) |
| `autoLaunch` | 开机自启 开/关 | 切换开关时 |
| `chromaColor`(预留) | 绿幕色值,默认标准绿 | 暂不暴露 UI |

读取需对缺字段容错,提供默认值。

## 6. 窗口行为

- 透明、无边框、置顶(`alwaysOnTop`)、不在任务栏显示(`skipTaskbar`)。
- 启动时读 `windowBounds` 还原上次位置和尺寸;读不到用默认(屏幕右下角,300×300)。
- **整窗可拖动**:CSS `-webkit-app-region: drag`,按住猫即可拖。
- **右键不拖动**:监听 `contextmenu` 事件单独弹菜单。左键拖、右键菜单,互不打架。
- 拖动停止 / 尺寸改变 → 防抖后写回 `windowBounds`。

## 7. 系统托盘 + 右键菜单

托盘图标与右键点猫弹出**同一套菜单**:

| 菜单项 | 行为 |
|---|---|
| 选择视频文件夹… | 系统弹窗选绿幕源文件夹,触发转换 + 重载 |
| 尺寸 ▸ | 子菜单:小(200) / 中(300) / 大(450) / 自定义…(弹输入框填像素) |
| 显示 / 隐藏 | 切换猫的可见性(托盘常驻,猫可临时藏起来) |
| 开机启动 ☐ | 勾选框,切换开机自启 |
| 退出 | 真正退出程序 |

## 8. 错误处理

| 情况 | 处理 |
|---|---|
| 没选过文件夹就启动 | 猫窗口显示提示「右键我 → 选择视频文件夹」,不报错 |
| 文件夹无视频 / 全不支持 | 托盘气泡提示「该文件夹没有可用视频」,保持上次状态或提示态 |
| 某视频 ffmpeg 转换失败 | 跳过该文件,继续转其余,日志记录,不中断整批 |
| 缓存中的透明视频被手动删 | 下次启动检测缺失,自动重转 |
| 视频列表为空 | renderer 显示占位提示,不黑屏、不崩 |
| 转换中程序被关 | 半成品作废:转完才写入最终文件名,避免半成品被当成成品 |

## 9. 测试策略

GUI + 系统集成程序,分两层:

**可单测的纯逻辑(Node 测试,不开窗口)**
- 视频文件扫描 + 过滤(只认 .mp4 / .mov / .webm / .mkv / .avi 等)
- 缓存判定逻辑(哪些需转、哪些命中缓存,按文件名 + 修改时间)
- 配置读写(默认值、缺字段容错)

**手动验证清单(GUI 部分)**
- 选绿幕文件夹 → 转换 → 猫出现且背景透明
- 一个播完自动切下一个、最后一个回到第一个循环
- 拖动、右键调尺寸、重启后位置尺寸都记住
- 托盘显示/隐藏、开机自启、退出

## 10. 项目文件结构

```
desktop_cat/
├── package.json
├── src/
│   ├── main.js              # 主进程:窗口/托盘/菜单/自启
│   ├── preload.js           # IPC 安全桥
│   ├── config.js            # 配置读写(可单测)
│   ├── video-scanner.js     # 扫描 + 过滤 + 缓存判定(可单测)
│   ├── converter.js         # ffmpeg 绿幕 → 透明 webm
│   └── renderer/
│       ├── index.html       # 透明播放器页面
│       ├── renderer.js      # 顺序循环播放逻辑
│       └── style.css
├── test/                    # 纯逻辑单测
└── assets/
    └── tray-icon.png        # 托盘图标
```

每个文件单一职责:`config` 管配置、`video-scanner` 管「有哪些视频要播/要转」、`converter` 管转格式、`renderer` 管播放。彼此通过明确的数据(路径数组、配置对象)通信,可各自独立理解与测试。
